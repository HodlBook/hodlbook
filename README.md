# HodlBook

This project aims to be a crypto management portfolio. 

## Initial features
- register assets deposits/withdrawals;
- register exchange from asset A to asset B;
- track current value of assets and value increase/decrease;
- show wallet share by asset;
- show wallet value and profit by currency of reference;

## wDecisions needed before actually starting development:
- back-end stack;
- front-end stack;
- deployment pipeline;
- deployment integration;

## What HodlBook doesn't want to be
- No automatic exchange API integrations
- No tax reporting or compliance features
- Not a hosted service or SaaS product

## Technical Direction
- Lightweight web application
- Docker Compose deployment
- Minimal dependencies (sqlite or bold databases)
- Self-contained as much as possible
